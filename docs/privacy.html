Privacy Policy

Last Updated: December 12, 2024

TL;DR: Acquisition Pulse is a privacy-first tool. Your business data stays on your machine. We don't collect, store, or share your order data, customer information, or financial metrics with anyone. Ever.

1. Overview

Acquisition Pulse ("we", "our", or "the application") is a self-hosted profitability analytics dashboard designed for e-commerce businesses. This Privacy Policy explains how we handle data when you use Acquisition Pulse.

2. Data We Do NOT Collect

Acquisition Pulse is built with a privacy-first architecture. We do not collect, transmit, or store:

  • Order data: Your Shopify CSV exports are processed locally on your machine and never transmitted to external servers
  • Customer information: Customer IDs, names, emails, addresses, or any personally identifiable information (PII) remain on your device
  • Financial metrics: Revenue, COGS, profit margins, LTV, CAC, and all calculated metrics stay local
  • Campaign data: Google Ads, Meta Ads, and TikTok Ads performance data is fetched via OAuth and stored locally
  • Usage analytics: We don't track how you use the dashboard, which features you access, or how often you log in
  • Authentication credentials: Passwords (if auth is enabled) are hashed with bcrypt and stored locally; we never see your plaintext password

3. Data Processing

3.1 Local Data Storage

When you upload a Shopify CSV or import data:

  • Data is processed in-memory using Python libraries (pandas, duckdb)
  • Temporary data may be stored in local application state during your session
  • No data is transmitted to external APIs or cloud services
  • Data persists only if you explicitly save dashboard state to a local file

3.2 OAuth Tokens

To connect Google Ads, Meta Ads, or TikTok Ads accounts, Acquisition Pulse uses OAuth 2.0 authentication:

  • What we store: OAuth access tokens and refresh tokens
  • Where they're stored: In your operating system's secure credential storage (macOS Keychain, Windows Credential Manager, Linux Secret Service)
  • Encryption: Tokens are encrypted at rest using your OS's native encryption
  • Access: Only the Acquisition Pulse application running on your machine can access these tokens
  • What we DON'T do: We never transmit OAuth tokens to our servers (because we don't have servers)

3.3 Google Ads API Usage

When you connect a Google Ads account:

  • Acquisition Pulse requests read-only access to campaign performance data
  • We fetch: campaign names, spend, impressions, clicks, conversions
  • We do NOT access: billing information, payment methods, or account settings
  • Data is pulled directly from Google's API to your local machine
  • Google may log API requests according to their own privacy policy

4. Third-Party Services

Acquisition Pulse integrates with third-party advertising platforms via their official APIs:

4.1 Google Ads API

  • Purpose: Fetch campaign spend data for profitability analysis
  • Data shared: None. We use OAuth to authenticate; Google does not receive your Shopify data
  • Google's Privacy Policy: https://policies.google.com/privacy

4.2 Meta Ads API (Future)

  • Purpose: Fetch Facebook/Instagram ad spend data
  • Data shared: None. OAuth authentication; no data transmitted to Meta beyond API requests
  • Meta's Privacy Policy: https://www.facebook.com/privacy/policy/

4.3 TikTok Ads API (Future)

  • Purpose: Fetch TikTok ad spend data
  • Data shared: None. OAuth authentication; no data transmitted to TikTok beyond API requests
  • TikTok's Privacy Policy: https://www.tiktok.com/legal/privacy-policy

5. Security Measures

We take data security seriously, even though your data never leaves your machine:

  • Encryption: OAuth tokens stored in OS keyring with native encryption
  • Password hashing: If authentication is enabled, passwords are hashed with bcrypt (industry-standard one-way hashing)
  • No plaintext storage: Sensitive credentials are never stored in plaintext
  • Local-only processing: All data analysis happens on your machine; no cloud uploads
  • Open source: Acquisition Pulse code is available for audit on GitHub

6. Data Retention

Since Acquisition Pulse is self-hosted and processes data locally:

  • You control retention: Data persists only as long as you keep it on your machine
  • Deletion: To delete data, simply remove uploaded CSV files or clear dashboard state
  • OAuth tokens: Revoke access via Google/Meta/TikTok account settings, then delete tokens from your OS keyring

7. Your Rights

Because your data stays on your device, you have full control:

  • Access: All your data is stored locally; you can view, export, or copy it anytime
  • Deletion: Delete files, clear browser cache, or uninstall Acquisition Pulse to remove all data
  • Portability: Export data in CSV, JSON, or other formats using built-in export features
  • Correction: Edit CSV files or re-upload corrected data at any time

8. Children's Privacy

Acquisition Pulse is designed for business use and is not intended for individuals under 18 years of age. We do not knowingly collect data from children.

9. Changes to This Privacy Policy

We may update this Privacy Policy from time to time. Changes will be posted on this page with an updated "Last Updated" date. If we make significant changes, we'll notify users via:

  • GitHub repository release notes
  • In-app notification (if applicable)
  • Email (if we add a mailing list in the future)

10. Contact Us

If you have questions about this Privacy Policy or how Acquisition Pulse handles data, please contact us:

  • Email: privacy@acquisitionpulse.com (or your contact email)
  • GitHub: Open an issue on the Acquisition Pulse repository

11. Compliance

Acquisition Pulse is designed to help you comply with data protection regulations:

11.1 GDPR (General Data Protection Regulation)

  • Acquisition Pulse acts as a data processor on your behalf
  • You (the business owner) are the data controller
  • Customer data in your Shopify CSV exports may include PII subject to GDPR
  • Since data stays local, you maintain full control and compliance responsibility

11.2 CCPA (California Consumer Privacy Act)

  • Acquisition Pulse does not "sell" data (we don't collect it in the first place)
  • You remain responsible for CCPA compliance for customer data in your Shopify exports

12. Google API Services User Data Policy

Acquisition Pulse's use of information received from Google APIs adheres to the Google API Services User Data Policy, including the Limited Use requirements.

Specifically:

  • We only request the minimum scopes necessary to fetch campaign performance data
  • We do not use Google user data for purposes unrelated to providing Acquisition Pulse functionality
  • We do not transfer Google user data to third parties (data stays local)
  • We do not use Google user data for advertising

13. Open Source Transparency

Acquisition Pulse is open source. You can review the source code to verify our privacy claims:

  • GitHub Repository: [Your GitHub URL]
  • License: [Your License, e.g., MIT]
  • All data processing logic is visible and auditable

Bottom line: Your data is yours. Acquisition Pulse is a tool you run on your machine to analyze your own data. We built it this way intentionally because we believe privacy is a fundamental right, not a feature.
